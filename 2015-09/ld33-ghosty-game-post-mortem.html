LD33: Ghosty Game post mortem

Published: 19 September, 2015

  • gamedev
  • post-mortem
  • ludum-dare

Results of Ludum Dare 33 got already published so it's time for postmortem.

My main goal - making the game in time for the Compo - has not been fulfiled. I was far from having anything ready to be send and the deadline has been reached. After getting some sleep, rest and spending time with a family I decided not to give up completely and try to finish anything for the Jam.

For these readers who have no clue what's the difference between LD compo and Jam, in really short summary:

  • Time to post entry to the compo is 48h, while for Jam it's 72h
  • You are only allowed to work alone in the Compo, but for Jam you might work in a team made of programmers, gfx artists, musicans, etc.
  • For Compo you must create all assets (graphics, audio) by yourself and all of these need to be made during the 48h of the Compo. For Jam you are allowed to use already existing assets.
  • Compo is very strict and limiting, Jam is more chilled out

The results are overall better. Keep in mind that in Jam you compete against teams of people; it's much harder to get better score due to this single fact. I was really suprised that my overall score was that high as the game was far from being finished. I've been slowly working on enhancing my graphics and audio skills and it seems like some progress has been made.

No humor and mood categories as I disabled them.

Timelapse

What went right

I won't repeat my goals as they might be found on the previous entry. Let's jump right into the right things!

Visuals

I'm really happy that my pixel-art skills got improved. I was able to create some animations and (for a beginner) decent looking characters. However, there is still much more to be improved. My next goal is to be able to produce content of similar quality to the Ultimate Fantasy tileset.

Audio

Good side of the audio was the music. I've used Bosca Ceoil program for the first time and it was very easy to create a simple tune. I think it's good idea to stick to this program for a now and focus on other fields.

Entity Component System

Before the LD I was working on my custom engine based on ECS architecture. I had no actual game to test if it works properly so LD was a first real test. There were nasty bugs that took some time from my developement, but overall I'm really happy with the architecture. If you start thinking with components and not game objects, everything magically starts working. If you got your rendering, physics and audio systems already prepared you might focus on actual gameplay because attaching proper components will make everything magically work!

What went wrong

Deadline missed

I was unable to deliver in time an entry for compo. I've lost over 6 hours of development due to a basic indexing issue which shouldn't happen to any software engineer. I didn't have any physic system ready to be used so I had to make one by myself. This combined with not tested ECS-based engine resulted in too much time spent in a debugger trying to figure out what's going on. Lesson (I hope) learnt and (I hope) next time I'll focus more on actual gameplay.

Lack of gameplay

As I've lost too much time fighting windmills, no planned gameplay feature has been implemented. My plan was to create a stealth evasion game with static obstacles, moving enemies with their cone of view, sound alarm system, and simple logic puzzle (push this button to activate that).
